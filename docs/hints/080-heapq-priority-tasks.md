# 080: 「急ぎの仕事から出てくる」のヒント

[問題](../problems/080-heapq-priority-tasks.md) / [解答](../solutions/080-heapq-priority-tasks.md)

**難易度:** ☆☆☆☆

??? tip "ヒント1"

    `heapq` は、リストを最小値が先に出るヒープとして扱います。
    `heappush` で追加し、`heappop` で最小の要素を取り出します。

??? tip "ヒント2"

    ヒープにタプルを入れると、先頭の要素から順に比較されます。
    `(priority, name)` では、同じ優先度のときにタスク名が比較されてしまいます。

??? tip "ヒント3"

    投入順を表す連番を持たせると、同じ優先度でも安定した順序で取り出せます。
    ヒープには `(priority, order, name)` を入れます。

??? tip "ヒント4"

    `__len__` を実装すると、`len(queue)` が使えるようになります。
    空のヒープから取り出す前に、自分で `IndexError` を送出するとエラーメッセージも指定できます。
